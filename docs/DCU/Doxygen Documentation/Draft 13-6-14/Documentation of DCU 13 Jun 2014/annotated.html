Data Structures
Here are the data structures with brief descriptions:
ACTIVITY_DETAILSThis structure will be used to store the details actvities to perform
ADAMAdam information for polling
BMS_DATA_INFOTo store details for sending data to BMS system. It contains BMS register address and the reference of storage pool where data / status is stored
BMS_INFOStores the details required for communication with BMS. BMS can be connected on TCP / serial. Also the protocol for communication with BMS can be configured
BMS_LISTThis structure stores the information regarding write request which will be eventually added to write queue
BMS_PROCESS_QUEUEDifferent information store for bms process queue over serial
BUFFERTo store details of buffer. Buffer will be used to transfer bytes read from the various connection / interfaces and pass on to the upper layer which will process it further
COMMUNICATION_INFOTo store communication related information
CONNECTIONTo store the details of each connection. Connection details are used to communicate on the interface
DCU_INFOStores the details of DCU. This information of DCU is used for resetting the SMS module. Also UI of the server / PC utility will use this for performaing specific actions on DCU
DEVICE_INFOTo store information related to all types of meters
DEVICE_REFTo store information related to to the devices based on the interface. This is to be used only after the sorting
ETH_INTERFACEStores the details of Ethernet interface. Used while registering interface
GPRS_INTERFACEStores the details of GPRS interface. Used while registering interface
HB_METHODSThis structure will be used when heartbeat repsonse will be parsed by the device. The device will parse the json formatted methods and fill the details in this structure, which will be used by the application to perform specific action
HB_STRUCTThis structure will be used when heartbeat repsonse will be parsed by the device. The device will parse the json formatted payload and fill the details in this structure, which will be used by the application to perform specific action
HTTP_JSON_PKT_DETAILTo store details of fill and parse function for each HTTP JSON packet
HTTP_PACKETStores information of entire HTTP packet in structure format
INFO_STRUCTThis structure is used by json parser to store the information related to key value and its length
INTERFACEStores the entire information of a particular interface
IO_PARAMSTo store information related to only IO devices. These are nothing to do about polling
JSON_COMM_CONTEXTTo store details of JSON communication over TCP based on various activities to be carried out
JSON_FILE_DOWNLOAD_STRUCTThis structure will be used when to store information of file download repsonse which will be parsed by the device. The device will parse the json formatted methods and fill the details in this structure, which will be used by the application to perform specific action
JSON_FILE_UPLOAD_STRUCTThis structure will be used when to store information of file upload repsonse which will be parsed by the device. The device will parse the json formatted methods and fill the details in this structure, which will be used by the application to perform specific action
JSON_PKT_HDR_STRUCTThis structure will be used to store information present in the packet. The device will parse the json formatted methods and fill the details in this structure, which will be used by the application to perform specific action
KEY_VALOFFSETThis structure is used by json parser to parse HB response. It contains key and the address to store value for that key
LISTTo store details of list. List will be used to transfer bytes read from the various connection / interfaces and pass on to the upper layer which will process it further
MB_RTUModbus RTU information for polling
MEDIA_INDEX_INFOTo store media related information based on the meter list
METER_PARAMSTo store information related to only meters. These are nothing to do about polling
MODBUS_REQ_INFOStores information required for polling the device connected over Modbus. This structure is used to store received data from device and for sending request over Modbus for particular device
POLLING_INFODevice polling information, this will be used for polling the data over Eth, Serial and also for Adam and smartmesh
SCH_METHODS
SCH_PENDING_DI_LIST
SCHEDULE
SCHEDULE_INFO
SERIAL_INTERFACEStores the details of serial interface. Used while registering interface
SERIAL_POLLING_INFOSerial polling information this will be use by serial threads for data polling and update the storage
SERVER_INTERFACEStores the details of server settings. Used while registering the interface
SMARTMESHModbus smartmesh information for polling
SYS_PARAMTo store the details of system parameters
SYSTEM_CFG_STRUCTStores the details of system configuration
TOKENSStores token information. It stores the reference of the start of token and length of token
WRITE_LISTIt contains ADAM protocol related information. It is used for sending write request to particular device connected over Serial
XML_KEY_VAL_STRUCTThis structure is used by xml parser to parse configuration files. It contains key and the address to store value for that key
